## Why TestWire?

You've written the controller. You've wired up the endpoints. Now you have to write 20 nearly identical test stubs before you can even start testing real logic.

TestWire reads your API and writes those stubs for you.

```bash
dotnet tool install -g testwire
testwire generate ./MyApi --output ./tests
```

```
✔ Found 3 controllers, 14 endpoints
✔ Generated 14 test stubs → ./tests/
```

## Usage

### Basic

```bash
testwire generate ./MyApi
```

Scans your project, outputs test files alongside your source by default.

### Custom output directory

```bash
testwire generate ./MyApi --output ./tests
```

### Target a single controller

```bash
testwire generate ./MyApi --controller ProductsController
```

### Options

| Flag | Description | Default |
|------|-------------|---------|
| `--output` | Output directory for generated tests | `./tests` |
| `--controller` | Target a specific controller | All controllers |
| `--framework` | Test framework (`xunit`, `nunit`, `mstest`) | `xunit` |
| `--dry-run` | Preview output without writing files | `false` |
